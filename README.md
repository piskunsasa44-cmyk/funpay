# BR Price Monitor v3 — мониторинг цен виртов Black Russia на FunPay + команды в Telegram

## Главное

Система работает так же, как раньше — **сама** проверяет цены по расписанию
и сама шлёт алерты в Telegram, без твоего участия. Команды ниже — это просто
ручной доступ к ней сверху, не замена автоматики.

- Отслеживаются **все серверы** (раньше было ограничение 1–30 — убрал)
- Интервал автопроверки по умолчанию **120 секунд** (было 180 — чаще проверяет)
- Лоты дешевле `EXTRA_CHEAP_THRESHOLD_RUB` (по умолчанию **19₽**) приходят с
  заголовком `❗❗❗ ОЧЕНЬ ДЁШЕВО ❗❗❗` вместо обычного

## Команды в Telegram

| Команда | Что делает |
|---|---|
| `/scan` | Проверить цены прямо сейчас, не дожидаясь автопроверки |
| `/top` или `/top 20` | Топ-N самых дёшевых лотов **прямо сейчас по всем серверам**, без привязки к порогу (по умолчанию 10, максимум 30) — для общей картины рынка |
| `/history` или `/history 20` | Последние алерты, которые уже были отправлены (по умолчанию 10, максимум 50) |
| `/status` | Текущий порог, работает/на паузе, когда была последняя проверка, сколько лотов нашлось |
| `/setthreshold 22` | Поменять порог алерта на лету, без передеплоя |
| `/pause` | Остановить автопроверку (можно всё равно вызывать `/scan` и `/top` вручную) |
| `/resume` | Включить автопроверку обратно |
| `/help` | Список команд |

Команды принимаются **только** из чата с твоим `TELEGRAM_CHAT_ID` — из других
чатов игнорируются и пишутся в лог.

## Переменные окружения

| Переменная | Обязательна | По умолчанию | Что это |
|---|---|---|---|
| `TELEGRAM_BOT_TOKEN` | да | — | Токен бота |
| `TELEGRAM_CHAT_ID` | да | — | Твой chat id |
| `FUNPAY_GOLDEN_KEY` | да | — | Cookie сессии FunPay |
| `PRICE_THRESHOLD_RUB` | нет | `26` | Порог алерта (можно менять и на лету через `/setthreshold`) |
| `EXTRA_CHEAP_THRESHOLD_RUB` | нет | `19` | Порог для пометки ❗❗❗ |
| `POLL_INTERVAL_SECONDS` | нет | `120` | Частота автопроверки. **Не ставь меньше 60** — риск временной блокировки по IP на стороне FunPay |
| `TELEGRAM_POLL_TIMEOUT` | нет | `10` | Long-poll таймаут получения команд (не трогай без причины) |
| `SANITY_MIN_PRICE_RUB` | нет | `5` | Защита от сломанной валюты/парсинга |
| `HISTORY_MAX_ENTRIES` | нет | `300` | Сколько последних алертов хранить для `/history` |
| `REALERT_COOLDOWN_SECONDS` | нет | `3600` | Не слать повтор по тому же лоту чаще раза в час |
| `DEBUG_ROWS` | нет | `0` | Отладочный дамп сырых строк таблицы в лог |

## Память между перезапусками

`seen_lots.json` (антиспам) и `history.json` (для `/history`) — файлы на диске
контейнера. Без подключённого Volume в Railway они сбрасываются при каждом
редеплое/перезапуске. Если хочешь, чтобы `/history` переживала рестарты —
**Settings → Volumes → New Volume**, примонтируй на `/app`.

## Проверка после деплоя

Напиши боту `/status` — должен прийти ответ с текущими настройками. Если
тишина больше минуты — смотри Deploy Logs, строка `Ошибка получения команд
из Telegram` подскажет причину (обычно неверный токен или сетевой сбой).

## Остальное (golden_key, отладка парсинга, Start Command)

Без изменений с прошлых версий: Custom Start Command — `python monitor.py`,
`golden_key` достаётся через F12 → Application → Cookies → funpay.com,
диагностика парсинга через `DEBUG_ROWS=5` в Variables.
