# BR Price Monitor — мониторинг цен виртов Black Russia на FunPay

Отдельный процесс. Не трогает твой основной Telegram-бот на Railway —
деплоится как второй, независимый сервис.

## Что делает

Раз в `POLL_INTERVAL_SECONDS` (по умолчанию 180с) проверяет страницу
валюты Black Russia на FunPay, фильтрует лоты по серверам 1–30 и шлёт
тебе в Telegram сообщение, если цена за 1kk ≤ 26₽ (или другого порога,
который ты задашь).

## Установка локально (для теста перед деплоем)

```bash
cd br-price-monitor
python3 -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## Переменные окружения (обязательные)

| Переменная | Что это | Как получить |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен бота | У @BotFather в Telegram — можешь использовать токен своего существующего бота или создать новый через `/newbot` |
| `TELEGRAM_CHAT_ID` | Куда слать алерты | Напиши своему боту любое сообщение, затем открой `https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates` в браузере — там будет `"chat":{"id": ...}` |

## Переменные окружения (опциональные)

| Переменная | По умолчанию | Зачем |
|---|---|---|
| `PRICE_THRESHOLD_RUB` | `26` | Порог цены за 1kk в рублях |
| `POLL_INTERVAL_SECONDS` | `180` | Частота опроса FunPay |
| `REALERT_COOLDOWN_SECONDS` | `3600` | Не слать повторный алерт по тому же лоту раньше, чем через час, если цена не упала ещё ниже |
| `FUNPAY_GOLDEN_KEY` | — | Cookie твоей сессии FunPay, если без авторизации цены показываются не в рублях. Смотри ниже как достать |

### Как достать `FUNPAY_GOLDEN_KEY`

1. Зайди на funpay.com в браузере под своим аккаунтом
2. F12 → вкладка Application (Chrome) / Storage (Firefox) → Cookies → funpay.com
3. Найди cookie с именем `golden_key`, скопируй значение
4. Не выкладывай это значение никуда публично — это ключ твоей сессии

## Запуск локально

```bash
export TELEGRAM_BOT_TOKEN="твой_токен"
export TELEGRAM_CHAT_ID="твой_chat_id"
python3 monitor.py
```

В логах увидишь что-то вроде:

```
2026-06-19 14:02:01 [INFO] Старт мониторинга. Порог=26₽/1kk, серверы=[1, 2, ..., 30], интервал=180с
2026-06-19 14:02:02 [INFO] Распарсено лотов: 47
2026-06-19 14:02:02 [INFO] Лотов ниже порога 26₽: 0
```

## Деплой на Railway

1. Создай новый сервис (отдельный от основного бота) → "Deploy from GitHub repo" или просто залей эту папку как отдельный репозиторий
2. В настройках сервиса добавь переменные окружения из таблицы выше
3. Railway сам подхватит `requirements.txt` и запустит `python monitor.py` —
   если нет, в Settings → Start Command укажи `python monitor.py`
4. Включи Restart Policy = "On Failure" — если процесс упадёт от сетевой
   ошибки (уже обработано в коде, но на всякий случай), Railway его поднимет заново

## Если парсер находит 0 лотов, хотя рынок живой

Значит FunPay поменял разметку страницы (`class` названия у блоков). Открой
страницу в браузере, нажми F12 → Elements, найди блок одного лота в таблице
и посмотри актуальные имена классов у элементов: сервер / продавец / количество
/ цена. Поправь селекторы в функции `parse_lots()` в `monitor.py` — это единственное
место, которое может потребовать правки при изменении вёрстки сайта.

## Важно

- Не ставь `POLL_INTERVAL_SECONDS` меньше 60 — слишком частые запросы могут
  привести к временной блокировке по IP на стороне FunPay
- Это парсинг публичной страницы лотов, без обхода каких-либо ограничений
  доступа — но учти, что автоматический сбор данных может не приветствоваться
  правилами площадки, поэтому держи частоту запросов разумной
